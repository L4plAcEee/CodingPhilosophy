Mistake

  1. 未处理 i 或 j 尚未遍历完的情况
    如果 a 和 b 长度不等，只处理了 i >= 0 && j >= 0 的交集，未处理剩余的部分。
  2. 进位处理不完整（k=0 的时候）
    当最高位有进位（如 999 + 1 = 1000），ret[0] 会是有效的 '1'，否则就是 '0'。但你在：
    ret[k] = '0' + carry;
    只写了一位，而没有处理后续进位（如果 carry == 1 且 k == 0，可能仍有问题）。
  3. 删除前导零可能导致空字符串
    如果结果是 "0"（比如 a = "0", b = "0"），你会删成空字符串，造成不合法的返回值。
